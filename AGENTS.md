Astro site built around Cloudflare Workers hosting, with SEO and i18n support. `main` is live production — every merge auto-deploys, treat changes as shipping to real users.

## Tech Stack

Framework: Astro 7, SSR (`output: "server"`)
Hosting: Cloudflare Workers via `@astrojs/cloudflare` + `wrangler` 4
UI: Svelte 5 (`@astrojs/svelte`)
Styling: Tailwind CSS 4 + daisyUI 5 — opt-in skill at `.agents/skills/daisyui/SKILL.md` (`@daisyui` or attach when writing component classes)
i18n: Paraglide JS 2 (`@inlang/paraglide-js`)
SEO / Head: `eminence-astro-suite` — opt-in skill at `.agents/skills/eminence-astro-suite/SKILL.md` (`@eminence-astro-suite` or attach for head/metadata work)
Sitemap: `@astrojs/sitemap` 3.7+ (older 3.0 lacks options this project relies on)
JSON-LD: `schema-dts`
Language: TypeScript 6, strict mode (`astro/tsconfigs/strict`)
Testing: Vitest 4
Package Manager: pnpm
Runtime: Node 22+
Formatting: Prettier 3 + astro/svelte/organize-imports plugins, enforced pre-commit via Husky + lint-staged

Path aliases (`tsconfig.json`): `@/*` → `./src/*`, `~/*` → `./public/*`.

## i18n — Paraglide (critical)

Never hardcode visible text; all text comes from Paraglide message functions, e.g. `m.page_heading()` imported from `@/paraglide/messages`.

Adding a message: add the key to `messages/es.json` first (base locale), then `messages/en.json`. The Vite plugin recompiles during `astro dev` / `astro build`.

Key naming: `page_section_element` snake_case, e.g. `about_hero_title`, `contact_form_submit`.

Runtime helpers from `@/paraglide/runtime`: `getLocale()` current locale, `localizeHref(path, { locale })` adds locale prefix, `deLocalizeHref(path)` strips it, `locales` array of configured locales.

Locale config: `es` is the base locale with no URL prefix, `en` is prefixed (`/en/...`). Strategy is `["url", "baseLocale"]` in `astro.config.mjs`. `src/middleware.ts` wraps every request in `paraglideMiddleware` and handles routing automatically — don't add manual locale-detection logic elsewhere.

## Layouts

Always wrap pages in a layout, never a bare page. Import `DefaultLayout` from `@/layouts/DefaultLayout.astro`, pass `title`/`description` from `m.page_title()`/`m.page_description()`.

`BaseLayout.astro` sets `html lang={getLocale()}`, imports global CSS and fonts. `DefaultLayout.astro` extends `BaseLayout` with Header, SkipToContent, main, Footer.

## Styling: Tailwind CSS 4 + daisyUI 5

Tailwind is configured in `src/styles/global.css` via `@import "tailwindcss"` (no `tailwind.config.js`, v4 is CSS-first). daisyUI is loaded as a CSS plugin with `dark` as the default theme. Prefer daisyUI semantic classes (`btn`, `card`, `badge`, `menu`, `dropdown`, etc.) over hand-rolled Tailwind utilities. Font is available as `font-sans` (Inter, via `astro.config.mjs` fonts + Google font provider, CSS variable `--astro-font-inter`).

## Svelte Components

Use Svelte 5 runes for interactive pieces. Always add a `client:*` directive in `.astro` files: `client:load` for critical interactivity, `client:idle` for lower priority, `client:visible` for below the fold. Pure display components with no interactivity should be `.astro` files instead of `.svelte`.

## Testing

Vitest tests live in `src/tests/**`, run with `pnpm test` or `pnpm test:watch`. For Astro components use `experimental_AstroContainer` (`astro:container`) to render to string and assert on output. New non-trivial logic needs at least one runnable test; trivial one-liners don't.

## SEO / Head — eminence-astro-suite

Configured in `astro.config.mjs` (currently only `icons.source: "src/assets/icon.svg"`). Use its head/meta components instead of hand-writing meta/link tags for title, description, Open Graph, canonical, robots, sitemap, manifest, etc. Use `schema-dts` for typed JSON-LD.

## Commits & Branching

Branch naming: `issue-number-type/brief-description`, e.g. `42-feat/add-contact-page`.

Commit format (Conventional Commits): `type(scope): description`. Type must match the branch's type and the issue's primary label.

Supported types: `feat` new feature or improvement, `fix` bug fix, `docs` documentation only, `refactor` quality/formatting/logic changes with no behavior change, `deps` dependency updates, `test` adding/correcting tests, `revert` reverting a commit (format `revert: <reverted commit header>`, body must include the SHA).

Exception: if a change doesn't fit any type, skip the `type(scope):` prefix and write a concise plain description instead, never exceeding 70 characters.

Always run `pnpm format` before committing (Husky + lint-staged enforce this automatically on staged files).
